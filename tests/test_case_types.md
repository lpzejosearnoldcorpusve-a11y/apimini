# Types of Test Cases

## 1. Functional Test Cases
These test cases verify that the application behaves as expected based on the functional requirements. Examples include:
- User login
- Profile update
- Search functionality

## 2. Non-Functional Test Cases
These test cases evaluate the performance, usability, and reliability of the application. Examples include:
- App loading time
- Responsiveness of the UI
- Compatibility with different devices

## 3. Regression Test Cases
These test cases ensure that new changes do not break existing functionality. Examples include:
- Verifying login after adding a new feature
- Checking map loading after updating the Maps screen

## 4. Integration Test Cases
These test cases verify the interaction between different modules of the application. Examples include:
- Interaction between the Maps screen and the Search bar
- Data flow between the Profile screen and the backend API

## 5. User Interface (UI) Test Cases
These test cases ensure that the UI elements are displayed and function correctly. Examples include:
- Verifying the alignment of buttons
- Checking the visibility of error messages

## 6. Security Test Cases
These test cases ensure that the application is secure from vulnerabilities. Examples include:
- Verifying password encryption
- Checking for SQL injection vulnerabilities

## 7. Usability Test Cases
These test cases ensure that the application is user-friendly. Examples include:
- Verifying the intuitiveness of the navigation
- Checking the readability of text on different screens

## 8. Performance Test Cases
These test cases evaluate the performance of the application under various conditions. Examples include:
- Testing the app under high user load
- Measuring the time taken to load the Maps screen
